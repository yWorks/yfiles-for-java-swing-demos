Description

The PartitionGrid demo application lets you experience the usage of a PartitionGrid for hierarchic and organic layout calculations.

Nodes are assigned to columns/rows by their fill/border color (black nodes are unassigned) and the calculated PartitionGrid is visualized after a layout run to show the effect the node assignment had for the layout.

The demo code configuring the PartitionGridData for the layout calculation can be found in the method PartitionGridConfigurator.createPartitionGridData.

Interaction

Calculating a new Layout

  • Clicking the Hierarchic or Organic button or pressing Ctrl/Command+H respectively Ctrl/Command+O configures the PartitionGrid and triggers the corresponding layout algorithm.
  • Note that the Organic layout does not support group nodes that contain child nodes assigned to different rows or columns unless Stretch group nodes is enabled.

Changing the node assignment

Right-clicking on a node opens a context menu with different options to change the assignment of this node to:
  • Remove grid restriction: The node is not assigned to a specific column or row but the layout algorithm may put it wherever it wants.
  • Switch column: The node is assigned to the same column as all other nodes having this fill color.
  • Switch row: The node is assigned to the same row as all other nodes having this border color.

Modifying the graph

  • Nodes and edges can be created and removed using the default gestures.
  • Group nodes can be created by selecting one or several nodes and clicking the Grouping button.

Configuring the PartitionGrid

  • Use fix column order: Whether the columns are ordered by the displayed colors or the layout algorithm may optimize the order.
  • Minimum column width: The minimum width of each column.

  • Stretch group nodes: Whether group nodes shall be stretched to span the grid cells of their child nodes or span only the child nodes themselves.

    Note that the cells that shall be spanned have to be declared before calculating the layout so the cells of unassigned (black) nodes are not considered.
