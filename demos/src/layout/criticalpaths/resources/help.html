Description

This demo shows how to emphasize important, or 'critical', paths with hierarchic and tree layout algorithms. (The algorithm may be changed in the demo source code.)
To make these critical paths more striking, nodes and edges in each path are aligned (in layout direction) if possible.
If there is more than one critical path, priorities can be assigned to define which path should be aligned in case a node or edge is part of more than one critical path.

Things to Try

  • Use the toolbar button to mark random paths in the graph and see how the layout changes.
  • Switch between examples for hierarchic layout and tree layout by changing the source code.
