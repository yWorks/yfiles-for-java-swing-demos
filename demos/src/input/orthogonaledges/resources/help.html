Description

Enable orthogonal edge editing using the GraphEditorInputMode. Customize the behavior by implementing the IOrthogonalEdgeHelper interface and adding instances to the edge decorator of the graph.

Things to try

Drag segments and bends of the edges and observe the different characteristics.

Edge Types

  • Red edges behave like normal edges with no orthogonal editing. The segments are straight lines with arbitrary slope. Dragging an unselected segment always creates a new bend.
  • Green edges demonstrate the default orthogonal edge editing behavior. While dragging, all segments stay orthogonal and new bends are created as needed.
  • Purple edges behave just like green edges except that the locations of the source port and target port are constraint to the given port candidates of the nodes.
  • Orange edges behave just like green edges except that the source and target port can be placed at any location inside the node since they use dynamic port candidates.
  • Blue edges behave just like green edges for the inner segments. The first and the last segment can have an arbitrary slope. If a new bend is added to one of these two segments, the newly created inner segment becomes orthogonal immediately.
